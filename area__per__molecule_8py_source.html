40 Notes
41  1) all selections are forced to be subsets of the initial selection. This is
42  necessary for the mapping of areas to work correctly.
43  2) this program assumes that the system has already been centered such that

48 Padding:
49 
50  This program uses the scipy Voronoi implementation, which is itself a
51  wrapper around Qhull. Qhull doesn't know anything about periodic boundary
52  conditions as far as I can tell, so we fake it by generating extra atoms so
53  provide an environment around the real ones; otherwise, you get insane
54  areas for the atoms at the "edge" of the box. You need to pick this value
55  such that there are ~2-3 layers of atoms around the real ones; in my
56  experience, a value of 15 ang works well if you're selecting all atoms or
57  all heavy atoms. If your selection is more sparse (e.g. just phosphate
58  atoms), you will need to make it significantly larger, perhaps 30 ang).
59  The best test is to run with different pad values on a short trajectory and
60  find the value where the answer doesn't change. Since a larger pad value
61  increases the number of atoms used in the Voronoi decomposition, it does
62  get more expensive, but it's never too bad, and there's always a faster way
63  to get a wrong answer.
64

68  To compute the distribution of areas/lipid molecule, where there are
69  2 kinds of lipids, one with segids L1, L2, ... L120, and the other with
70  segids L121, L121, ... you could do 3 selections like
71 
72  'segid =~ "^L\d+" && \!hydrogen' 'segid -> "^L(\d+)" <= 120' 'segid -> "^L(\d+)" > 120'
73 
74 Note that for a bilayer you have to think carefully about the z-range you use.
75 If you use a z-range of [0, 20], you'll get the upper leaflet accurately, but
