118 string fullHelpMessage(void)
119  {
120  string msg =
121 "\n"
122 "SYNOPSIS\n"
123 "\n"
124 "Compute the distribution of a membrane physical property about a membrane\n"
125 "protein.\n"
126 "\n"
127 "DESCRIPTION\n"
128 "The purpose of this tool is to compute one of a number of physical \n"
129 "properties on a 2D grid surrounding a membrane protein. The properties \n"
130 "number density, height, molecular order parameter, and orientation vector,\n"
131 "although the code is written to make it easy to add other quantities (see \n"
132 "below). The system is aligned against the coordinates of the selection \n"
133 "specified with --align-selection using the first unskipped frame as \n"
134 "reference, unless the --ref-structure option is given, in which case \n"
135 "the file specified there is used (--align-selection is still applied)\n"
136 "The alignment is performed in two dimensions, so that the \n"
137 "lipid bilayer is not tilted or shifted; it is assumed that the bilayer\n"
138 "normal is the z-axis, and that the bilayer center is at z=0\n"
139 "\n"
140 "By default, the target is treated as a single large entity. The --splitby\n"
141 "flag will let you break it up into individual molecules, segments, or \n"
142 "residues.\n"
143 "\n"
144 "Options\n"
145 "--calc The type of calculation to be performed.\n"
146 " density: number density of the selection\n"
147 " height: average z-position of the centroid of the selection\n"
148 " order: molecular order parameter (see below)\n"
149 " vector: orientation vector\n"
150 "\n"
151 " The molecular order parameter is calculated using the \n"
152 " principal axes of the selection; the 2nd and 3rd axes are\n"
153 " averaged, and plugged into the standard 0.5 (3 cos^2 - 1) \n"
154 " formula, relative the z-axis. If the selection is a lipid\n"
155 " chain, then the values are comparable to the ones seen for\n"
156 " lipid order parameters.\n"
157 "\n"
158 " The orientation vector is average of the xy components of the\n"
159 " first principal axis of the selection. Unlike the other \n"
160 " options, which return scalars, this returns a 2D vector, \n"
161 " which can be plotted in gnuplot using the \"with vector\" \n"
162 " option.\n"
163 "\n"
164 "\n"
165 "\n"
166 "EXAMPLE\n"
167 "\n"
168 "membrane_map --xmin -30 --xmax 30 --ymin -30 --ymax 30 --xbins 30 --ybins 30 --splitby mol example.psf dark_ensemble_20.dcd 'segid == \"RHOD\"' 'resname == \"DHA\"'\n"
169 "\n"
170 " This sets the histograms to run from -30:30 in x and y, with \n"
171 " 2 ang x 2 ang bins. It uses the segment name RHOD to align the\n"
172 " snapshots, and uses DHA chains as the targets. Since no \n"
173 " calculation type is specified, a number density is calculated. \n"
174 " The DHA chains are split up on the basis of connectivity.\n"
175 "\n"
176 "POTENTIAL COMPLICATIONS\n"
177 "\n"
178 "The code will break if the alignment and target selections overlap, \n"
179 "because the 2D alignment works by setting the z-coordinates of the \n"
180 "alignment selection to 0. \n"
181 "\n"
182 "In regions where there's no data (e.g. inside the region occluded by \n"
183 "whatever you're aligning to), the code outputs a value of -999999999.\n"
184 "chosen as the ANSI standard insane value. The exception is if you're\n"
185 "doing a density calculation, in which case there's no divide by zero\n"
186 "and sanity reigns everywhere.\n"
187 "\n"
188 "The options --upper-only and --lower-only let you calculate properties\n"
189 "using only the upper and lower leaflets respectively. The check is done\n"
190 "for each frame, so these options will handle the case where a component\n"
191 "is capable of flipping between leaflets on the MD timescale. However,\n"
192 "the implementation assumes that the membrane has been previously centered\n"
193 "at z=0. For obvious reasons, you can't specify both --upper-only and \n"
194 "--lower-only at the same time. Note: the stated number of matching target\n"
195 "molecules output at the beginning of the run does not take this restriction\n"
196 "into account.\n"
197 "\n"
198 "IMPLEMENTING NEW QUANTITIES\n"
199 "\n"
200 "Implementing new quantities is quite easy as long as they return either\n"
201 "a scalar (double) or a vector (GCoord). You begin by editing \n"
202 "membrane_map.hpp -- all of the classes that do the work are subclasses of \n"
203 "CalcProperty, so you'll need to create a new class analogous to the ones\n"
204 "already there, e.g. CalcDensity or CalcMolOrder. You'll need to supply\n"
205 "a constructor that calls the CalcProperty constructor and a method called\n"
206 "calc that will do the calculation. The last step of the calc method must\n"
207 "be calling the incr function, which will add the value into the histogram.\n"
208 "If the quantity being calculated is a density, you may need to supply a \n"
209 "normalize method as well (see CalcDensity for an example).\n"
210 "\n"
211 "Once the class is written, you just have to hook it in so that the binary\n"
212 "knows about it. To do this, edit membrane_map.cpp. First, you'll need to\n"
213 "add the new type to the enum CalcType (near the top of membrane_map.cpp).\n"
214 "Second, edit the postConditions method in the ToolOptions class to add \n"
215 "your calculation type (it's just a string of if-else-ifs). Third, \n"
216 "edit the switch statement in the main body of the code to add your \n"
217 "new method for calculating (there's only 1 switch). Finally, \n"
218 "if you intend anyone else to use your method, edit the documentation \n"
219 "string in addGeneric (also in ToolOptions) and the fullHelpMessage \n"
220 "function.\n"
221 "\n"
222  "\n";
223  return(msg);

229 int main(int argc, char *argv[])
230  {
231 
232  string hdr = invocationHeader(argc, argv);
233  cout << "# " << hdr << endl;
234 
235  opts::BasicOptions* bopts = new opts::BasicOptions(fullHelpMessage());
236  opts::TrajectoryWithFrameIndices* tropts = new opts::TrajectoryWithFrameIndices;
237  opts::BasicSplitBy *sopts = new opts::BasicSplitBy;
238  opts::RequiredArguments* ropts = new opts::RequiredArguments;
239  ToolOptions* topts = new ToolOptions;
240  ropts->addArgument("align-selection", "selection to align on");
241  ropts->addArgument("target-selection", "selection to calculate with");
242 
243  opts::AggregateOptions options;
244  options.add(bopts).add(tropts).add(ropts).add(topts).add(sopts);
245  if (!options.parse(argc, argv))
246  {
247  exit(-1);
248  }
249 
250  AtomicGroup system = tropts->model;
251  pTraj traj = tropts->trajectory;
252  vector<uint> frames = tropts->frameList();
253  traj->readFrame(frames[0]);
254  traj->updateGroupCoords(system);
255 
256  AtomicGroup align_to = selectAtoms(system, ropts->value("align-selection"));
257 
258  AtomicGroup reference;
259  if ((topts->reference_filename).length() > 0)
260  {
261  AtomicGroup reference_system = createSystem(topts->reference_filename);
262  reference = selectAtoms(reference_system, ropts->value("align-selection"));
263  }
264  else
265  {
266  reference = align_to.copy();
267  }
268 
269  AtomicGroup apply_to = selectAtoms(system, ropts->value("target-selection"));
270 
271  vector<AtomicGroup> targets = sopts->split(apply_to);
272  cout << "# Found " << targets.size() << " matching molecules" << endl;
273 
274  // Set up storage for our property.
275  double xmin = topts->xmin;
276  double xmax = topts->xmax;
277  double ymin = topts->ymin;
278  double ymax = topts->ymax;
279  uint xbins = topts->xbins;
280  uint ybins = topts->ybins;
281  double xwidth = (xmax - xmin)/xbins;
282  double ywidth = (ymax - ymin)/ybins;
283 
284  CalcPropertyBase *calculator;
285  switch (topts->type)
286  {
287  case DENSITY:
288  calculator = new CalcDensity(xbins, ybins, xwidth, ywidth);
289  break;
290  case ORDER:
291  calculator = new CalcMolOrder(xbins, ybins);
292  break;
293  case HEIGHT:
294  calculator = new CalcHeight(xbins, ybins);
295  break;
296  case VECTOR:
297  calculator = new CalcOrientVector(xbins, ybins);
298  break;
299  default: // this can't happen, set in option handling
300  cerr << "ERROR: unknown calculation type" << endl;
301  exit(-1);
302  }
303 
304  // We don't want the transformation to tilt the membrane, so we'll
305  // zero out the z coordinates before using the alignment.
306  // We'll do the same
307  for (AtomicGroup::iterator i = reference.begin();
308  i!= reference.end();
309  ++i)
310  {
311  (*i)->coords().z() = 0.0;
312  }
313 
314  // loop over frames in the trajectory
315  for (uint i=0; i<frames.size(); ++i)
316  {
317  traj->readFrame(frames[i]);
318  traj->updateGroupCoords(system);
319 
320 
321  // zero out the alignment selections z-coordinate
322  AtomicGroup align_to_flattened = align_to.copy();
323  for (AtomicGroup::iterator j = align_to_flattened.begin();
324  j!= align_to_flattened.end();
325  ++j)
326  {
327  (*j)->coords().z() = 0.0;
328  }
329 
330 
331  // get the alignment matrix
332  GMatrix M = align_to_flattened.superposition(reference);
333  M(2,2) = 1.0; // Fix a problem caused by zapping the z-coords...
334  XForm W(M);
335 
336  // align the stuff we're goign to do the calculation on
337  apply_to.applyTransform(W);
338 
339  // Calculate something
340  uint xbin, ybin;
341  for (vector<AtomicGroup>::iterator j = targets.begin();
342  j != targets.end();
343  ++j)
344  {
345  GCoord centroid = j->centroid();
346  // Skip molecules outside the xy range of interest
347  if ( (centroid.x() < xmin) || (centroid.x() > xmax) ||
348  (centroid.y() < ymin) || (centroid.y() > ymax)
349  )
350  {
351  continue;
352  }
353  // If the user chose to look at only one leaflet,
354  // skip molecules in the opposite leaflet.
355  // Note: this assumes that the membrane is centered at z=0
356  else if ((centroid.z() > 0) && topts->lower_only)
357  {
358  continue;
359  }
360  else if ((centroid.z() < 0) && topts->upper_only)
361  {
362  continue;
363  }
364  else
365  {
366  xbin = (uint)(centroid.x() - xmin) / xwidth;
367  ybin = (uint)(centroid.y() - ymin) / ywidth;
368  }
369 
370  // do the work
371  calculator->calc(*j, xbin, ybin);
372  }
373 
374  }
375 
376  // loop over bins and dump out the values
377  calculator->normalize(frames.size());
378 
379  cout << "# X\tY\tValue(s)" << endl;
380  for (uint i = 0; i < xbins; ++i)
381  {
382  double xval = xmin + xwidth*i;
383  for (uint j = 0; j < ybins; ++j)
384  {
385  double yval = ymin + ywidth*j;
386 
387  cout << xval << "\t"
388  << yval << "\t"
389  << calculator->print(i,j)
390  << endl;
391  }
392  cout << endl;
393  }
394  }
