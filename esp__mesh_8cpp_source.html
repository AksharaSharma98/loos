16 string fullHelpMessage(void) {
17  string msg =
18  "SYNOPSIS \n"
19 " Fit charges on a mesh composed of spherical shells to reproduce the \n"
20 " electrostatic potential due to solvent around a solute. \n"
21 " \n"
22 "DESCRIPTION \n"
23 " The goal of this tool is to replace solvent configurations sampled from\n"
24 " a trajectory with a mesh of point charges representing the mean field \n"
25 " electrostatic contribution of the solvent-solute interaction. A common \n"
26 " application is producing a set of point charges to represent solvent \n"
27 " approximately in a QM/MM calculation. The idea comes from the iPolQ \n"
28 " method for deriving aqueous phase partial charges [1]. \n"
29 " \n"
30 " Given an atom selection representing a solute, this tool will generate\n"
31 " a mesh of points defined by a union of spherical shells centered on the\n"
32 " coordinates of those atoms in the first frame of the trajectory. \n"
33 " A mesh of equidistant points on a spherical shell with user-specified \n"
34 " radius and point density is generated using a using a Fibonacci \n"
35 " spiral [2]. For each atom, these points are rotated to a random \n"
36 " orientation using quaternions [3,4] generated using a random point on \n"
37 " the surface of a 4D hypersphere [5] and then translated to the \n"
38 " coordinates of the atom. Then, points closer than the specified radius \n"
39 " to another atom are removed. The result is a mesh of points surrounding\n"
40 " and equidistant to the solute atoms. \n"
41 " \n"
42 " Two such meshes are generated at different radii. This tool will \n"
43 " compute the classical electrostatic potential (ESP) due to solvent \n"
44 " molecules at each atom in the selection and at each point in the closer\n"
45 " mesh. Solvent molecules contribute to the ESP only if at least one atom\n"
46 " is closer than a cutoff distance (-C option) but no atom is closer than\n"
47 " the farther radius to any solute atom. Then, point charges are placed \n"
48 " at points on the farther mesh, and their magnitude is fit by linear \n"
49 " regression to reproduce the solvent ESP, averaged over the trajectory, \n"
50 " at the points on the closer mesh. The sum of square errors between the \n"
51 " ESP at the atoms in the selection due to these point charges and that \n"
52 " due to the solvent informs on the quality of the fit. \n"
53 " \n"
54 " Multiple meshes at different radii can be used for the fit point \n"
55 " charges. The farthest mesh will be fit first. Then, subsequent meshes \n"
56 " will fit the difference between the trajectory-averaged solvent ESP and\n"
57 " the ESP due to the fit point charges on farther meshes. The radii are \n"
58 " specified in the -R option by passing a string of numbers in descending\n"
59 " order separated by spaces. \n"
60 " \n"
61 " The first line of the output is the command-line input. The second line\n"
62 " contains the number of points on the closer ESP mesh, the number of \n"
63 " points on the farther charge mesh, the sum of the fit point charges, \n"
64 " and the sum of square errors from the charge fit. The third line \n"
65 " contains the ESP at the solute atoms due to solvent averaged over the \n"
66 " trajectoy. The fourth line contains the ESP at the solute atoms due to \n"
67 " the fit point charges. Then, for each point on the closer mesh, there \n"
68 " will be a line containing the coordinates, the ESP due to solvent \n"
69 " averaged over the trajectory, and the ESP due to the fit point charges.\n"
70 " Finally, the output contains the coordinates and magnitude of each fit \n"
71 " point charge. Units for charges are fundamental charge (e), and units \n"
72 " for ESP are kcal mol^-1 e^-1. \n"
73 " \n"
74 " The default selection for the solvent is any residue named \"WAT\" or \n"
75 " ending in \"+\" or \"-\". This will catch things like \"Cl-\" or \n"
76 " \"Ca2+\" but not \"HOH\" or \"Mg\". The default selection for the \n"
77 " solute is any residue not named \"WAT\" or ending in \"+\" or \"-\". \n"
78 " However, the solute and solvent selections do not need to be exactly \n"
79 " complementary. \n"
80 " \n"
81 " If a solvent other than water is used, the maximum number of atoms per \n"
82 " solvent molecule needs to be set using the --solvent_size option. This \n"
83 " number needs to be known to allocate memory for a matrix of distances \n"
84 " between solvent atoms and mesh points. \n"
85 " \n"
86 " This tool makes no attempt to assess the convergence of the trajectory.\n"
87 " Users should check that the solvent ESP is converged with respect to \n"
88 " the trajectory length, the sampling interval, and the size of the \n"
89 " solvent box. \n"
90 " \n"
91 "EXAMPLES \n"
92 " esp_mesh foo.pdb foo.dcd \n"
93 " Fit a charge mesh at 10 angstroms to reproduce the solvent ESP at \n"
94 " 3 angstroms for a solute solvated in water and salt. \n"
95 " \n"
96 " esp_mesh -R 4 -S 2.5 foo.pdb foo.dcd \n"
97 " Fit a charge mesh at 4 angstroms to reproduce the solvent ESP at \n"
98 " 2.5 angstroms for a solute solvated in water and salt. \n"
99 " \n"
100 " esp_mesh -R '10 8 6' foo.pdb foo.dcd \n"
101 " Fit three charge meshes at 10, 8, and 6 angstroms to reproduce the \n"
102 " solvent ESP at 3 angstroms. \n"
103 " \n"
104 " esp_mesh -C 40 foo.pdb foo.dcd \n"
105 " Fit a charge mesh at 10 angstroms to reproduce the solvent ESP at \n"
106 " 3 angstroms. Only solvent molecules between 10 and 40 angstroms will \n"
107 " contribute to the solvent ESP. \n"
108 " \n"
109 " esp_mesh -f foo_esp.dat foo.pdb foo.dcd \n"
110 " Dump a time series of the solvent ESP at the closer mesh to foo_esp.dat\n"

113 " -w \"resname == 'WAT' || resname == 'MOH'\" foo.pdb foo.dcd \n"

116 " \n"
117 " esp_mesh -c foo.pdb foo.dcd \n"
118 " Fit a charge mesh allowing solvent molecules in between the closer ESP \n"

129 " \n"
130 "REFERENCES \n"

220 // Generate equidistant points on a spherical shell using a Fibonacci spiral
221 // Swinbank R & Purser RJ (2006) Q J R Meteorol Soc 132, 1769
222 // N: number of points
223 // sphere_grid: flattened vector of grid coordinates of size 3 * N
224 void generateEquidistantSphereGrid(const uint N, vector<double>& sphere_grid) {
225 
226  double z; // z = cos(theta), where theta is the polar angle from the +z axis
227  double sin_theta; // sin(theta)
228  double phi; // azimuthal angle from the +x axis
229 
230  // Interval between z coordinates of points
231  const double z_interval = 2.0 / N;
232  // Interval between phi coordinates of points. Equal to
233  // 2 * pi * (1 - 1 / golden_ratio), where golden_ratio is (1 - sqrt(5)) / 2
234  const double golden_angle = M_PI * (3 - sqrt(5));
235 
236  // Generate the grid
237  z = 1.0 + z_interval * 0.5;
238  phi = -golden_angle * 0.5;
239  for (uint i = 0; i < N; ++i) {
240  // Even distribution of z values from +1 to -1
241  // Equivalent to z = 1 - 2 / N * (i + 0.5)
242  z -= z_interval;
243  // Rotate around the z axis by goldenAngle radians
244  // Equivalent to phi = goldenAngle * (i + 0.5)
245  phi += golden_angle;
246  // z = cos(theta) and sin^2(theta) + cos^2(theta) = 1
247  sin_theta = sqrt(1 - z * z);
248  // Convert spherical coordinates to Cartesian coordinates
249  sphere_grid.push_back(sin_theta * cos(phi));
250  sphere_grid.push_back(sin_theta * sin(phi));
251  sphere_grid.push_back(z);
252  }
253 
254 } // generateEquidistantSphereGrid()

256 // Generate a random rotation using a quaternion (q1, q2, q3, q0)
257 // uniform_pm_one: function to produce uniform random numbers on (-1, 1)
258 // rotation_matrix: random rotation matrix in three dimensions
259 void generateRandomRotation(RNG& uniform_pm_one,
260  vector<double>& rotation_matrix) {
261 
262  double r1, r2; // Random numbers uniform on (-1, 1)
263  double s1 = 2.0, s2 = 2.0; // Sums of squares of two random numbers
264  double q1, q2, q3, q0; // Quaternion components
265  // Intermediate quantities for construction of rotation matrix
266  double q1_2, q2_2, q3_2, q0_2, q1_q2, q2_q3, q3_q1, q1_q0, q2_q0, q3_q0;
267 
268  // Generate a random point on a 4D hypersphere efficiently
269  // Marsaglia G (1972) Ann Math Stat 43, 645
270  while (s1 >= 1.0) {
271  q1 = uniform_pm_one();
272  q2 = uniform_pm_one();
273  s1 = q1 * q1 + q2 * q2;
274  }
275  while (s2 >= 1.0) {
276  r1 = uniform_pm_one();
277  r2 = uniform_pm_one();
278  s2 = r1 * r1 + r2 * r2;
279  }
280  q3 = r1 * sqrt((1.0 - s1) / s2);
281  q0 = q3 * r2 / r1;
282 
283  // Construct the rotation matrix in terms of the quaternion
284  // Quaternion components in terms of Euler angles (phi, theta, psi) are
285  // q1 = sin(theta / 2) * cos((phi - psi) / 2)
286  // q2 = sin(theta / 2) * sin((phi - psi) / 2)
287  // q3 = cos(theta / 2) * sin((phi + psi) / 2)
288  // q0 = cos(theta / 2) * cos((phi + psi) / 2)
289  // Vesely FJ (1982) J Comp Phys 47, 291
290  // Frenkel D & Smit B (2002) Understanding Molecular Simulation, 2e, 49
291  q1_2 = q1 * q1;
292  q2_2 = q2 * q2;
293  q3_2 = q3 * q3;
294  q0_2 = q0 * q0;
295  q1_q2 = 2.0 * q1 * q2;
296  q2_q3 = 2.0 * q2 * q3;
297  q3_q1 = 2.0 * q3 * q1;
298  q1_q0 = 2.0 * q1 * q0;
299  q2_q0 = 2.0 * q2 * q0;
300  q3_q0 = 2.0 * q3 * q0;
301  rotation_matrix[0] = q1_2 - q2_2 - q3_2 + q0_2;
302  rotation_matrix[1] = q1_q2 - q3_q0;
303  rotation_matrix[2] = q3_q1 + q2_q0;
304  rotation_matrix[3] = q1_q2 + q3_q0;
305  rotation_matrix[4] = q2_2 - q3_2 - q1_2 + q0_2;
306  rotation_matrix[5] = q2_q3 - q1_q0;
307  rotation_matrix[6] = q3_q1 - q2_q0;
308  rotation_matrix[7] = q2_q3 + q1_q0;
309  rotation_matrix[8] = q3_2 - q1_2 - q2_2 + q0_2;
310 
311 } // generateRandomRotation()

313 // Generate grid of equidistant points on a surface defined by a union of
314 // spherical shells centered on atoms in a selection
315 // uniform_pm_one: function to produce uniform random numbers on (-1, 1)
316 // grid_atom_coords: coordinates used to center the spherical shells
317 // N_atom: number of atoms in the selection
318 // radius: radius of the spherical shells
319 // N_shell: number of points per spherical shell
320 // surface_grid: flattened vector of the surface grid coordinates
321 void generateSurfaceGrid(RNG& uniform_pm_one,
322  const vector<GCoord>& grid_atom_coords, const uint N_atom,
323  const double radius, const uint N_shell, vector<GCoord>& surface_grid) {
324 
325  vector<double> sphere_grid; // Grid on a spherical shell
326  vector<double> rotation_matrix(9); // Rotation matrix
327  double atom_x, atom_y, atom_z; // Coordinates of an atom
328  double sphere_x, sphere_y, sphere_z; // Coordinates of point in sphere_grid
329  double grid_x, grid_y, grid_z; // Coordinates of point in surface_grid
330  double dx, dy, dz; // Components of a vector from an atom to a grid point
331  const double R_2 = radius * radius; // Radius squared
332  bool record_grid_point; // Whether to include a grid point in surface_grid
333  uint jj; // Index into sphere_grid
334 
335  // Generate grid of equidistant points on a spherical shell
336  sphere_grid.reserve(3 * N_shell);
337  generateEquidistantSphereGrid(N_shell, sphere_grid);
338 
339  // For each atom, rotate sphere_grid to a random orientation, expand to
340  // radius, and translate to the coordinates of the atom. Then, delete grid
341  // points within a distance of R from any other atom.
342  for (uint i = 0; i < N_atom; ++i) {
343 
344  atom_x = grid_atom_coords[i][0];
345  atom_y = grid_atom_coords[i][1];
346  atom_z = grid_atom_coords[i][2];
347 
348  // Generate random rotation matrix in three dimensions
349  generateRandomRotation(uniform_pm_one, rotation_matrix);
350 
351  for (uint j = 0; j < N_shell; ++j) {
352 
353  jj = 3 * j;
354  sphere_x = sphere_grid[jj];
355  sphere_y = sphere_grid[jj + 1];
356  sphere_z = sphere_grid[jj + 2];
357 
358  // Apply random rotation, expand to radius, and translate
359  // grid = (rotation_matrix @ sphere) * radius + atom
360  grid_x = (rotation_matrix[0] * sphere_x + rotation_matrix[1] *
361  sphere_y + rotation_matrix[2] * sphere_z) * radius + atom_x;
362  grid_y = (rotation_matrix[3] * sphere_x + rotation_matrix[4] *
363  sphere_y + rotation_matrix[5] * sphere_z) * radius + atom_y;
364  grid_z = (rotation_matrix[6] * sphere_x + rotation_matrix[7] *
365  sphere_y + rotation_matrix[8] * sphere_z) * radius + atom_z;
366 
367  // Check whether grid point is closer than radius to any other atom
368  record_grid_point = true;
369  for (uint k = 0; k < N_atom; ++k) {
370 
371  // Don't check current atom
372  if (k == i) continue;
373 
374  // Get components of vector from atom to grid point
375  dx = grid_x - grid_atom_coords[k][0];
376  dy = grid_y - grid_atom_coords[k][1];
377  dz = grid_z - grid_atom_coords[k][2];
378 
379  // Check whether grid point is closer than R
380  if (dx * dx + dy * dy + dz * dz < R_2) {
381  record_grid_point = false;
382  break;
383  }
384 
385  } // Loop over atoms, k
386 
387  // Record grid point if it is farther than radius from other atoms
388  if (record_grid_point) {
389  surface_grid.push_back(GCoord(grid_x, grid_y, grid_z));
390  }
391 
392  } // Loop over grid points, j
393 
394  } // Loop over atoms, i
395 
396 } // generateSurfaceGrid()

398 // Calculate electrostatic potential (ESP) at points in esp_grid and at atoms in
399 // grid_atoms due to solvent molecules in which no atom is within a low cutoff
400 // but at least one atom is within a high cutoff of any atom in grid_atoms. The
401 // low cutoff is given by fit_radius[i], and the high cutoff is given by
402 // fit_radius[i - 1] for i > 1 and by high_cutoff for i == 0.
403 // solvent_molecules: molecules in the solvent that generate the ESP
404 // N_solvent: number of solvent molecules
405 // solvent_size: largest number of atoms in a solvent molecule
406 // grid_atoms: atoms at which the ESP will be evaluated
407 // N_atom: number of atoms in grid_atoms
408 // esp_grid: grid points at which the ESP will be evaluated
409 // N_esp_points: number of grid points
410 // solvent_boundary: boundaries on solvent distances to grid atoms
411 // N_region: number of solvent-containing regions
412 // solvent_esp: ESP due to solvent molecules
413 // close_solvent: include solvent closer than the closest surface
414 void calculateFrameESP(vector<AtomicGroup>& solvent_molecules,
415  const uint N_solvent, const uint solvent_size,
416  const AtomicGroup& grid_atoms, const uint N_atom,
417  const vector<GCoord>& esp_grid, const uint N_esp_points,
418  const vector<double>& solvent_boundary, const uint N_region,
419  rMatrix& solvent_esp, const bool close_solvent) {
420 
421  double R2_min; // Closest squared distance between solvent and grid atoms
422  double solvent_charge; // Charge of a solvent atom
423  GCoord solvent_coords; // Coordinates of a solvent atom
424  uint surface; // Index of fitting surface
425 
426  // Matrix of squared distances between solvent atoms and grid atoms
427  rMatrix R2(solvent_size, N_atom);
428 
429  // Loop over solvent molecules
430  for (uint i = 0; i < N_solvent; ++i) {
431 
432  // Compute squared distances between all solvent atoms and all grid
433  // atoms and find the minimum of such squared distances
434  R2_min = solvent_boundary[0];
435 
436  // Loop over solvent atoms
437  for (uint j = 0; j < solvent_molecules[i].size(); ++j) {
438 
439  solvent_coords = solvent_molecules[i][j]->coords();
440 
441  // Loop over grid atoms
442  for (uint k = 0; k < N_atom; ++k) {
443  R2(j,k) = solvent_coords.distance2(grid_atoms[k]->coords());
444  if (R2(j,k) < R2_min)
445  R2_min = R2(j,k);
446  }
447  }
448 
449  // Identify to which region this solvent molecule belongs
450 
451  // Exclude solvent molecules farther than the cutoff
452  if (R2_min >= solvent_boundary[0])
453  continue;
454 
455  // Loop over solvent boundaries
456  for (uint j = 1; j <= N_region; ++j) {
457  if (R2_min >= solvent_boundary[j]) {
458  surface = j - 1;
459  break;
460  }
461  }
462 
463  // Solvent molecule closer than the closest fitting surface
464  if (R2_min < solvent_boundary[N_region]) {
465  if (close_solvent && R2_min >= solvent_boundary[N_region + 1])
466  surface = N_region - 1; // Include in ESP for closest surface
467  else
468  continue; // Exclude
469  }
470 
471  // Calculate ESP due to this solvent molecule
472 
473  // Loop over solvent atoms
474  for (uint j = 0; j < solvent_molecules[i].size(); ++j) {
475 
476  solvent_charge = solvent_molecules[i][j]->charge();
477  solvent_coords = solvent_molecules[i][j]->coords();
478 
479  // Loop over grid points in esp_grid
480  for (uint k = 0; k < N_esp_points; ++k)
481  solvent_esp(surface, k) += solvent_charge /
482  solvent_coords.distance(esp_grid[k]);
483 
484  // Loop over atoms in grid_atoms
485  for (uint k = 0; k < N_atom; ++k)
486  solvent_esp(surface, k + N_esp_points) += solvent_charge /
487  sqrt(R2(j,k));
488 
489  }
490 
491  } // Loop over solvent molecules, i
492 
493 } // calculateFrameESP()

495 // Write the solvent ESP at esp_grid and at grid atoms to an output file
496 // esp_file: name of output file
497 // frame: number of frames processed so far
498 // index: index of current frame
499 // solvent_esp: ESP due to solvent molecules
500 void writeESP(const string& esp_file, const uint frame, const uint index,
501  const rMatrix& solvent_esp) {
502 
503  ofstream esp_out(esp_file.c_str(), ios::app); // Stream for output file
504  double esp_sum; // Sum of ESP for different regions
505 
506  esp_out << index;
507 
508  // Loop over columns, i.e. points in esp_grid and grid_atoms
509  for (uint i = 0; i < solvent_esp.cols(); ++i) {
510 
511  esp_sum = 0;
512 
513  // Loop over rows, i.e. fitting surfaces
514  for (uint j = 0; j < solvent_esp.rows(); ++j)
515  esp_sum += solvent_esp(j, i);
516 
517  // Print esp due to solvent in all regions averaged over the number of
518  // frames processed so far

527 // Fit the magnitude of points charges located at grid points in fit_grid to
528 // reproduce the solvent ESP at grid points in esp_grid using linear regression
529 // from the LAPACK routine DGELS
530 // one_over_R: fitting matrix containing reciprocal distance
531 // target_esp: fitting target containing solvent ESP
532 // N_esp: number of grid points at which ESP was evaluated
533 // fit_charges: magnitudes of point charges to reproduce solvent ESP
534 // N_fit: number of point charges
535 // weight: weight of harmonic restaint of fit charges to zero
536 void fitChargeGrid(const rMatrix& one_over_R, const vector<double>& target_esp,
537  const uint N_esp, vector<double>& fit_charges, const uint N_fit,
538  const double weight) {
539 
540  // Number of rows in fitting matrix
541  uint N_row;
542  if (weight == 0)
543  N_row = N_esp;
544  else
545  N_row = N_esp + N_fit;
546 
547  // Arrays passed to Fortran routines must be in column-major order
548  Math::Matrix<double, Math::ColMajor> target(N_row, 1); // Fitting target
549  Math::Matrix<double, Math::ColMajor> matrix(N_row, N_fit); // Fitting matrix
550 
551  // Copy target_esp to target and copy oneOverR to matrix because they will
552  // be overwritten by DGELS
553  for (uint i = 0; i < N_esp; ++i) {
554  target[i] = target_esp[i];
555  for (uint j = 0; j < N_fit; ++j)
556  matrix(i, j) = one_over_R(i, j);
557  }
558 
559  // Add harmonic restraints to restrain magnitude of fit charges to zero
560  if (weight > 0)
561  for (uint i = 0; i < N_fit; ++i)
562  matrix(i + N_esp, i) = weight;
563 
564  // Set up arguments to DGELS
565  char TRANS = 'N';
566  int M = N_row;
567  int N = N_fit;
568  int NRHS = 1;
569  int LDA = M;
570  int LDB = M;
571  double PRE_WORK;
572  int LWORK = -1;
573  int INFO;
574 
575  // First call to DGELS to get value of LWORK
576  dgels_(&TRANS, &M, &N, &NRHS, matrix.get(), &LDA, target.get(), &LDB,
577  &PRE_WORK, &LWORK, &INFO);
578  if (INFO != 0)
579  throw(NumericalError("DGELS failed to estimate value of LWORK."), INFO);
580  LWORK = (int) PRE_WORK;
581  double* WORK = new double[LWORK+1];
582 
583  // Second call to DGELS to solve linear regression
584  dgels_(&TRANS, &M, &N, &NRHS, matrix.get(), &LDA, target.get(), &LDB, WORK,
585  &LWORK, &INFO);
586  if (INFO != 0)
587  throw(NumericalError("DGELS failed to solve linear regression."), INFO);
588 
589  // Now target contains the solution vector (magnitudes of fit charges) in
590  // the first N_fit elements, and the sum of square residuals (RSS) is the
591  // sum of the squares of elements N_fit + 1 to N_esp
592  double sum_Q = 0;
593  for (uint i = 0; i < N_fit; ++i) {
594  sum_Q += target[i];
595  fit_charges.push_back(target[i]);
596  }
597  double RSS = 0;
598  for (uint i = N_fit; i < N_row; ++i)
599  RSS += target[i] * target[i];
600  cout << boost::format("# %4d %4d %12.8f %14.8e\n") % N_esp % N_fit % sum_Q %
601  RSS;
602 
603 } // fitChargeGrid()
