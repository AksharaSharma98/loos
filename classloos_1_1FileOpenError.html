Detailed Description

Error while opening a file. 

Proxy of C++ loos::FileOpenError class.

This mostly represents issues with actually opening the file, such as a bad filename, permissions, as well as scanning the file as part of instantiation.

Some classes will automatically read either the whole file or, in the case of trajectories, the first frame. This can result in a FileReadError exception being thrown, despite being part of a logical "open" operation.
