45 const string msg =
46 "SYNOPSIS \n"
47 " \n"
48 "This tool is designed to find trajectories that have distorted structures or \n"
49 "bad rewrapping by checking for overlong bonds in some frame within the \n"
50 "trajectory. There are two modes of operation. In the first, the tool scans all \n"
51 "frames until it finds a problem, whereupon it either returns 1 (EXIT_FAILURE) \n"
52 "if it has, or 0 (EXIT_SUCCESS) if not. The second additionally writes a time \n"
53 "series to a provided file name listing each violating bond for each frame. \n"
54 "Note that merge-traj with --fix-imaging may be able to resolve issues flagged\n"
55 "by this program. gmx-trjconv with cluster imaging may also work.\n"
56 " \n"
57 "DESCRIPTION \n"
58 " \n"
59 "The tool uses predefined connectivity to look through all frames for bonds that\n"
60 " are longer than some cutoff within the user-specified selection. If it is \n"
61 "operating in default mode, that is if no time-series is requested, it will exit\n"
62 " upon finding the first such flawed bond. It will either return an \n"
63 "'EXIT_FAILURE' if it has found at least one, or 'EXIT_SUCCESS' if not. It can \n"
64 "also write a time-series of these bonds to a filename provided by the user, in \n"
65 "which case it will check all frames and report any flawed bonds to the file \n"
66 "name provided. For processing large datasets the first mode is likely more \n"
67 "helpful and will almost certainly be faster. \n"
68 " \n"
69 "The intention is for the return value to be used in bash control flow to allow \n"
70 "a user to conditionally add a trajectory file to a list of files with problems \n"
71 "(probably PBC issues, but I don't know your life), or perhaps operate on them \n"
72 "directly if the tool flags them (see 'EXAMPLES' below). This is analogous to \n"
73 "using 'grep -q' (quiet mode) to ascertain whether a regex is somewhere inside a\n"
74 " file, then operating conditionally in response to whether it is present or \n"
75 "absent. The tool provides a quiet mode to support this scripting style. The \n"
76 "time-series mode of operation can be nice as a way to spot check where the \n"
77 "issue is, since any frames noted there can then be looked up in a visualizer \n"
78 "and inspected manually to see what is really going wrong. The the return-value \n"
79 "of the program works the same whether a time-series is requested or not. \n"
80 " \n"
81 "If a time-series filename is provided as an argument to the option, a comment-\n"
82 "line containing the invocation is written to the first line. Subsequent lines \n"
83 "have four values separated by a space, each representing a bond that is \n"
84 "overlong: the frame index, the first atomID within the pair of atoms \n"
85 "constituting some bond, the second atomID in that bond, and that bond's \n"
86 "calculated length. Only non-redundant ID pairs in an order-independent fashion \n"
87 "are checked or reported (since atomID 2 bonded to atomID 7 is the same as 7 \n"
88 "bonded to 2). \n"
89 " \n"
90 "EXAMPLES \n"
91 " \n"
92 "The most basic mode of operation for this tool is: \n"
93 " \n"
94 "long-bond-finder model.psf traj.dcd \n"
95 "if [[ $? > 0 ]]; then \n"
96 " echo this traj is goofed \n"
97 "fi \n"
98 " \n"
99 "More useful is the situation where this is used in some loop, silencing the \n"
100 "per-traj operations using output redirection: \n"
101 " \n"
102 "for traj in xtcs/*.xtc; do \n"
103 " if ! long-bond-finder model.psf $traj 1 >> logfile.log 2>&1; then \n"
104 " echo $traj is goofed. 'DO SOMETHING!!!' \n"
105 " echo $traj >> list_of_goofy_trajs.txt \n"
106 " mv $traj goofy_trajs/ \n"
107 " fi \n"
108 "done \n"
109 " \n"
110 "If no log-file is desired, throw the '--quiet' flag to suppress emission of the\n"
111 " invocation header. \n"
112 " \n"
113 "for traj in xtcs/*.xtc; do \n"
114 " if ! long-bond-finder --quiet model.psf $traj; then \n"
115 " echo $traj is goofed. 'DO SOMETHING!!!' \n"
116 " echo $traj >> list_of_goofy_trajs.txt \n"
117 " mv $traj goofy_trajs/ \n"
118 " fi \n"
119 "done \n"
120 " \n"
121 " \n"
122 "Running either of these two commands with the addition of the '--timeseries \n"
123 "filename.dat' flag will write the four-column timeseries of bond violations \n"
124 "(possibly none) to 'filename.dat'. \n"
125 " \n"
126 "To infer connectivity for some model, provide the cutoff distance for a 'bond' \n"
127 "as an argument to the option: \n"
128 " \n"
129 "long-bond-finder --infer-connectivity 1.9 my_minimal_model.pdb traj.dcd \n"
130 " \n"
131 "To change what the cutoff for distorted bonds is, use the '--max-bond value' \n"
132 "flag. For example, if a 3.2 Angstrom cutoff were desired: \n"
133 " \n"
134 "long-bond-finder --max-bond 3.2 model.psf traj.dcd \n"
135 " \n"
136 "POTENTIAL COMPLICATIONS \n"
137 " \n"
138 "Note that as with all loos tools, trajectory file indexes are zeros based, but \n"
139 "some visualizers can be ones-based. In addition, GROMACS-generated trajectories\n"
140 " sometimes have initial coords saved to the 'first' frame, leaving the \n"
141 "possibility that the literal value in the first column could be off by either \n"
142 "one or two from what is displayed in a visualizer, depending on the \n"
143 "circumstances. For issue diagnosis purposes this seems OK, and the zeros-based \n"
144 "index is correct if one wanted to write a secondary script that used those \n"
145 "values as indexes into the trajectory to do something to goofed up frames. \n"
146 " \n"
147 "This tool doesn't currently use periodicity for bond length calculations, in \n"
148 "part because it was written to spot bad wrapping issues from harder to wrap \n"
149 "systems with periodicity that is not presently supported in loos (rhombic \n"
150 "dodecahedra, for example). Because many loos tools do use periodicity for \n"
151 "distance calculations, this could surprise some users. It seems hard to catch \n"
152 "bad wrapping with a distance calculation if that calculation respected PBCs. \n"
153 " \n"
154 "Also note that although a mechanism is provided to use models that don't have \n"
155 "connectivity, this option should be deployed cautiously. It uses a simple \n"
156 "distance cutoff to deduce where chemical bonds likely are for the model as a \n"
157 "whole based on the coordinates in the first frame. This will be incorrect if \n"
158 "the first frame has bonds that are overlong (or extreme collisions) relative to\n"
159 " the user's expectation, and while the tool is likely to find something \n"
160 "objectionable in the successive screwed up mess that its output will become, \n"
161 "manual inspection would be needed to be sure that the objection was not a false\n"
162 " positive. Using models with chemical connectivity based on a more reliable \n"
163 "source, such as a system-specifying file from an MD engine, is highly advised. \n"
164 " \n"
165 "Finally, note that the time-series mode could produce unwieldy output in the \n"
166 "case where a system specifying file is fundamentally damaged (say the thing \n"
167 "that is wrong with it is that its bond indices are off by one for all atoms). \n"
168 "The potential for absurdly voluminous output can be guarded against by checking\n"
169 " whether a trajectory has major issues by not writing the time-series as a \n"
170 "first past, then if a trajectory is flagged visualizing its contents to ensure \n"
171 "that the time-series won't have a ridiculous number (possibly nearly all) of \n"
172 "bonds to record for each frame. Similarly, using the customary loos trajectory \n"
173 "flags (range) alongside the time-series flag could allow one to only consider \n"
174 "the report on the first few frames, then inspecting these for pathology which \n"
175 "would make seeing the rest of the time-series irrelevant and overwhelming. \n"
176 ;

219 int main(int argc, char *argv[]) {
220 
221  string header = invocationHeader(argc, argv);
222  opts::BasicOptions *bopts = new opts::BasicOptions(msg);
223  opts::BasicSelection *sopts = new opts::BasicSelection("all");
224  opts::TrajectoryWithFrameIndices *tropts =
225  new opts::TrajectoryWithFrameIndices;
226  ToolOptions *topts = new ToolOptions;
227 
228  opts::AggregateOptions options;
229  options.add(bopts).add(sopts).add(tropts).add(topts);
230 
231  if (!options.parse(argc, argv))
232  exit(-1);
233 
234  // set up system for looping. Load coords from frame 0 into scope.
235  AtomicGroup model = tropts->model;
236  if (model.hasBonds()) {
237  } else if (topts->bondlength > 0)
238  if (model.hasCoords())
239  model.findBonds(topts->bondlength);
240  else {
241  throw(LOOSError(
242  "Model does not have coordinates with which to infer connectivity.\n"
243  ));
244  }
245  else
246  throw(LOOSError(
247  "Model does not appear to have chemical connectivity, and "
248  "infer-connectivity has not been set to a positive value.\n"));
249  AtomicGroup scope = selectAtoms(model, sopts->selection);
250  pTraj traj = tropts->trajectory;
251  traj->updateGroupCoords(model);
252  // should be a vector of two-atom AGs, each a pair of atoms in a bond
253  vector<AtomicGroup> bond_list = scope.getBondsAGs();
254  // set threshold for length of an unacceptable bond.
255  const double max_bond2 = topts->max_bond * topts->max_bond;
256 
257  // Operating in scanning mode;
258  // don't report anything except the presence of an unacceptable bond
259  if (topts->timeseries.empty()) {
260  // if thrown, don't even write invocation to stdout
261  if (!topts->quiet)
262  cout << "# " << header << "\n";
263  for (auto frame_index : tropts->frameList()) {
264  traj->readFrame(frame_index);
265  traj->updateGroupCoords(scope);
266  for (auto b : bond_list) {
267  if (b[0]->coords().distance2(b[1]->coords()) > max_bond2) {
268  return EXIT_FAILURE;
269  }
270  }
271  }
272  } else {
273  // Operating in timeseries mode;
274  // write a timeseries to file name provided by user
275  ofstream tsf(topts->timeseries);
276  bool found_viol = false;
277  tsf << "# " << header << "\n"
278  << "# frame atomID1 atomID2 bondlength\n";
279  for (auto frame_index : tropts->frameList()) {
280  traj->readFrame(frame_index);
281  traj->updateGroupCoords(scope);
282  float dist2 = 0;
283  for (auto b : bond_list) {
284  dist2 = b[0]->coords().distance2(b[1]->coords());
285  if (dist2 > max_bond2) {
286  found_viol = true;
287  tsf << frame_index << " " << b[0]->id() << " " << b[1]->id()
288  << " " << sqrtf(dist2) << "\n";
289  }
290  }
291  }
292  if (found_viol)
293  return EXIT_FAILURE;
294  }
295 
296  return EXIT_SUCCESS;
297 }
