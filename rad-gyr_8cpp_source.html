40 const string fullHelpMessage =
41  // clang-format off
42  "SYNOPSIS \n"
43 " \n"
44 "Read a trajectory and return a histogram of the radius of gyration of a \n"
45 "selection. \n"
46 " \n"
47 "DESCRIPTION \n"
48 " \n"
49 "This tool computes the radius of gyration for a selection of atoms for each \n"
50 "frame of a provided multi-trajectory. By default this selection is treated as \n"
51 "one group, but if --by-molecule is thrown then the selection will be split by \n"
52 "connectivity. The skip, stride, and range options operate on the multi-\n"
53 "trajectory as they would for other LOOS tools. The time-series option specifies\n"
54 " a file to write a timeseries of the radius of gyrations to. If none is \n"
55 "specified then no time-series is written. The num-bins, min-bin, and max-bin \n"
56 "options determine the extent and bin-width of the histogram, which is written \n"
57 "to stdout. The histogram contains both the probability per bin, and the \n"
58 "cumulative probability. \n"
59 " \n"
60 "EXAMPLE \n"
61 " \n"
62 "rad-gyr -k 100 -n 20 -m 5 -M 25 --by-molecule model.pdb traj1.dcd traj2.dcd \n"
63 "traj3.dcd \n"
64 " \n"
65 "This will concatenate traj1, traj2, and traj3 into one virtual trajectory, skip\n"
66 " the first 100 frames of each, and then histogram the radius of gyration of all\n"
67 " the atoms in each of the molecules in the model provided with the \n"
68 "trajectories, computing their radii of gyration and summing over them. Note \n"
69 "that this would be nonsensical if the molecules in the model were not multiple \n"
70 "copies of the same molecule, since all such Rgyr will be collated into one \n"
71 "histogram at the end. Using selection to pick a subsystem of interest makes \n"
72 "more sense in the context of most analyses of a solute. \n"
73 " \n"
74 "POTENTIAL COMPLICATIONS \n"
75 " \n"
76 "Gromacs users should be aware that this code assumes the system's\n"
77 "molecules are not split across the periodic boundary. If that's your\n"
78 "use-case, apply different reimaging first before using this tool.\n"
79 "LOOS provides facility to solve these problems in 'merge-traj' and \n"
80 "subsetter.\n"
81 "\n"
82 "The by-molecule flag applies splitByMolecule to the whole system, which is then\n"
83 "subset on a per molecule basis by the selection string provided. This means \n"
84 "that if you had a three chain protein and you selected 'name == \"CA\"', you'd \n"
85 "get three different groups of alpha carbons, each of which would be used to \n"
86 "calculate a separate radius of gyration. Most of the time this is probably what\n"
87 " you want, but if you wanted to use your selection to break up whatever is in \n"
88 "your system into fragments, then track each of those fragments separately, \n"
89 "throw the --by-fragment flag instead. Doing so flips the order in which the \n"
90 "selection string and the call to splitByMolecule are applied (in the by-\n"
91 "fragments case the selection is performed first, then the splitting by \n"
92 "connectivity) to the model provided. Thus if you provided a discontinuous \n"
93 "selection like the above for alpha carbons, then you'd be tracking the radii of\n"
94 " gyration of all those discontinuous fragments as though they were separate \n"
95 "molecules. \n"
96 "\n"
97 "It's also worth saying that this tool requires connectivity if either of the \n"
98 "splitting flags are thrown. You could use PyLOOS to add connectivity to your \n"
99 "system by reading it in, adding connectivity (for example, with findBonds) then\n"
100 " writing it back out again. Caution is advised, though; if you can use a model \n"
101 "that has connectivity from a better source, like a force field for example (for\n"
102 " example, prmtops and psfs both have this) you should prefer that to distance-\n"
103 "based workarounds like findBonds. \n"
104 " \n"
105 "Finally, if you are using one of the molecule splittings, know that you will be\n"
106 " accumulating all the various radii of gyration your selection is tracking for \n"
107 "each molecule it found in the histogram the tool reports. This, again, is \n"
108 "probably what you want. However, it could produce subtly screwy results if, for\n"
109 " example, you were tracking a molecule you did not believe you were tracking, \n"
110 "or if a molecule you thought you were tracking was actually being treated as \n"
111 "two molecules. As with all LOOS tools, it's a good idea to use model-select and\n"
112 " the pdb option to visualize what atoms any selection string under \n"
113 "consideration tracks before applying a tool that will give necessarily opaque \n"
114 "output from a goofed up selection. This warning is included here because it \n"
115 "might be particularly hard to spot an erroneous selection in a histogram of a \n"
116 "complicated system, like all the lipids in a membrane. \n"
117 ;

166 // Do the work of recording, either in the case where a TS is requested,
167 inline void histogram_rgyr(vector<greal> &hist, const greal rgyr,
168  const greal min_dist, const greal max_dist,
169  const greal bin_width, int &count,
170  ofstream &outfile) {
171  if ((rgyr >= min_dist) && (rgyr < max_dist)) {
172  hist[int((rgyr - min_dist) / bin_width)]++;
173  count++;
174  }
175 }
176 inline void histogram_molecules_rgyr(vector<greal> &hist,
177  vector<AtomicGroup> &molecules,
178  const greal min_dist, const greal max_dist,
179  const greal bin_width, int &count,
180  const int frame, ofstream &outfile) {
181  for (auto molecule : molecules) {
182  histogram_rgyr(hist, molecule.radiusOfGyration(), min_dist, max_dist,
183  bin_width, count, outfile);
184  }
185 }
186 // or in the case when not.
187 inline void ts_hist_rgyr(vector<greal> &hist, vector<AtomicGroup> &molecules,
188  const greal min_dist, const greal max_dist,
189  const greal bin_width, int &count, const int frame,
190  ofstream &outfile) {
191  greal rgyr;
192  outfile << frame;
193  for (auto molecule : molecules) {
194  rgyr = molecule.radiusOfGyration();
195  outfile << "\t" << rgyr;
196  histogram_rgyr(hist, rgyr, min_dist, max_dist, bin_width, count, outfile);
197  }
198  outfile << "\n";
199 }

201 int main(int argc, char *argv[]) {
202  string header = invocationHeader(argc, argv);
203 
204  opts::BasicOptions *bopts = new opts::BasicOptions(fullHelpMessage);
205  opts::BasicSelection *sopts = new opts::BasicSelection("all");
206  opts::MultiTrajOptions *mtopts = new opts::MultiTrajOptions;
207  ToolOptions *topts = new ToolOptions;
208 
209  opts::AggregateOptions options;
210  options.add(bopts).add(sopts).add(mtopts).add(topts);
211  if (!options.parse(argc, argv))
212  exit(-1);
213 
214  // Write histogram results to stdout. Write timeseries, if asked for, to file
215  cout << "# " << header << "\n";
216  ofstream tsf(topts->timeseries);
217  // make a function pointer with a signature matching the
218  void (*frameOperator)(vector<greal> & hist, vector<AtomicGroup> & molecules,
219  const greal min_dist, const greal max_dist,
220  const greal bin_width, int &count, const int frame,
221  ofstream &outfile);
222 
223  // establish system, and molecular subsystems
224  vector<AtomicGroup> molecules;
225  if (topts->by_molecule)
226  molecules = mtopts->model.splitByMolecule(sopts->selection);
227  else if(topts->by_fragment)
228  molecules = selectAtoms(mtopts->model, sopts->selection).splitByMolecule();
229  else
230  molecules.push_back(selectAtoms(mtopts->model, sopts->selection));
231 
232  // pick which operation to perform per frame using function pointer
233  if (topts->timeseries.empty())
234  frameOperator = histogram_molecules_rgyr;
235  else {
236  tsf << "# " << header << "\n"
237  << "# frame";
238  // Label each column with the starting and ending index for the 'molecule'
239  // as a stand in for the case where molecule's chainids are inaccurate.
240  for (auto molecule : molecules) {
241  int first = molecule[0]->index();
242  int last = (*(molecule.end() - 1))->index();
243  tsf << "\tatoms" << first << "-" << last;
244  }
245  tsf << "\n";
246  frameOperator = ts_hist_rgyr;
247  }
248  // prepare for trajectory loop
249  // counter for number of molecules in histogram bounds
250  int count = 0;
251  const int num_bins = topts->num_bins;
252  const greal min_dist = topts->min_dist;
253  const greal max_dist = topts->max_dist;
254  const greal bin_width = (max_dist - min_dist) / num_bins;
255 
256  // define and zero histogram
257  vector<greal> hist(num_bins, 0.0);
258  while (mtopts->trajectory->readFrame()) {
259  mtopts->trajectory->updateGroupCoords(mtopts->model);
260  // call fxn pointer to accumulate histogram and optionally write timeseries
261  (*frameOperator)(hist, molecules, min_dist, max_dist, bin_width, count,
262  mtopts->trajectory->currentFrame(), tsf);
263  }
264  // Write the histogram to stdout
265  cout << "# Rgyr\tProb\tCum" << endl;
266  greal cum = 0.0;
267  for (int i = 0; i < num_bins; i++) {
268  greal d = bin_width * (i + 0.5) + min_dist;
269 
270  greal prob = hist[i] / count;
271  cum += prob;
272  cout << d << "\t" << prob << "\t" << cum << endl;
273  }
274  tsf.close();
275 }
