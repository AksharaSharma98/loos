37  This program computes the X-ray scattering from a trajectory using the
38  spherical scattering approximation:
39 
40  I(q) = (sum over atom pairs) Fi(q) Fj(q) sin qd / qd
41  where F(q) are the atomic scaterring profiles and d is the distance
42  between the atoms.
43 
44  Command line:
45  scattering.py system_file selection out_file traj_file
46  system_file: the file describing system content (eg PDB, PSF)
47  selection: loos selection string out atoms to do the scattering
48  calculation on
49  out_file: file containing the output (see below)
50  traj_file: trajectory file
51 
52  Other options:
53  --qmin: minimum q value used. Default=0 The intensity curve is
54  normalized by the I at this value
55  --qmax: maximum q value. Default=6.0
56  --nq: number of q values to use. Default=20
57  --deduce_element: If specified, attempt to deduce the element of each
58  atom from the atom's name. THIS IS NOT VERY ROBUST, and will
59  give incorrect guesses for atoms named "CAL" (calcium) or
60  "SOD" (sodium). If possible, avoid using this option in favor
61  of specifying the system with a file format that gives us the
62  element or the mass (eg PSF). It should get proteins, RNA, lipids,
63  and water correct.
64  --skip: skip this number of frames from the front of the trajectory,
65  default=0
66  --stride: step through the trajectory by this number, default=1
67 
68  Output: The resulting data file is field-delimited text. The first column
69  is the q value. Second is I(q)/I(0), the normalized intensity. Third
70  is q*Rg, where Rg is the radius of gyration. Fourth column is
71  (q*Rg)^2 I(q)/I(0). Plotting 1 vs 2 gives a standard intensity
72  plot (I recommend log scale), while 3 vs 4 gives a Kratky plot
73 
74  Note: although the guts of this code are in C++, it's still doing a double
75  loop over all pairs of atoms (including self), times the number of q values,
76  for each frame. Think carefully about what you want to include in the
77  calculation, since this will probably be pretty slow for an explicit atom
78  system.
79

81  no effort to account for solvent structure unless the water is explicitly
82  included in the calculation. If you want a program that does that, consider

104 def deduceAtomicNumber(system):
105  """
106  Given an AtomicGroup system, guess the atomic number based on the atom's
107  name. NOT a particularly robust method, but useable if your system file
108  doesn't give you the element.
109  Only knows hydrogen, carbon, nitrogen, oxygen, sulfur. This covers
110  protein, nuclei acid, and lipid, but will probably fail on salt (calcium and
111  sodium will probably clash)
112  """
113 
114  for atom in system:
115  n = atom.name()
116  if n.startswith("H"):
117  atom.atomic_number(1)
118  elif n.startswith("C"):
119  atom.atomic_number(6)
120  elif n.startswith("N"):
121  atom.atomic_number(7)
122  elif n.startswith("O"):
123  atom.atomic_number(8)
124  elif n.startswith("P"):
125  atom.atomic_number(15)
126  elif n.startswith("S"):
127  atom.atomic_number(16)

179  formFactors = loos.FormFactorSet()
180 
181  total = np.zeros([num_qvals])
182  q_vals = np.arange(q_min, q_max, (q_max - q_min)/num_qvals)
183  rgyr = 0.0
184  for frame in traj:
185  total += np.asarray(subset.scattering(q_min, q_max, num_qvals,
186  formFactors))
187  rgyr += subset.radiusOfGyration()
188 
189  total /= total[0] # output I/I(0)
190  rgyr /= len(traj)
191 
192  qrg = q_vals * rgyr
193  kratky = qrg * qrg * total
