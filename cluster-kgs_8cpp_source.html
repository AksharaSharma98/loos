24 std::string fullHelpMessage(void) {
25  std::string helpstr =
26  "usage: \n"
27 "cluster-kgs < similarity_scores.asc > clustering_results.json \n"
28 " \n"
29 "cluster-kgs mimics the clustering aspect of the NMRCLUST utility that is \n"
30 "incorporated as part of the OLDERADO webserver for structural biology \n"
31 "informatics. It was originally published as: \n"
32 "Kelly, Gardner, and Sutcliffe, Prot. Eng. 9 11 1063-1065 (1996) \n"
33 "This type of clustering exists in other places, most notably in R, and has been\n"
34 " put to many uses beside clustering protein structures with their RMSD as the \n"
35 "distance between each structure. It is called cluster-kgs because this method \n"
36 "is referred to in other contexts (that is, where it is not being used to \n"
37 "analyze NMR ensembles) as kgs clustering, and because this executable operates \n"
38 "on a provided similarity matrix it is similarly flexible. Note that we do not \n"
39 "implement the 'eigen analysis' for cluster center determination, instead \n"
40 "choosing to use the element from each cluster with the lowest mean distance to \n"
41 "the other elements in the cluster. \n"
42 " \n"
43 "The tool works by reading in a similarity score matrix from a file (or stdin) \n"
44 "and writing the clustering results to stdout. The results report the index of \n"
45 "each cluster, with all the elements in each cluster following its index on the \n"
46 "same line. It will also provide an exemplar (the element nearest the centroid) \n"
47 "for each cluster in a separate block. The input matrix should be an NxN \n"
48 "symmetric matrix of similarity scores where the ij-th element is the similarity\n"
49 " between datum i and datum j. The similarity score matrix is expected to be \n"
50 "whitespace delimited in the column and newline delimited in the row. '#' is an \n"
51 "acceptable comment character, but only produces a comment-read at the beginning\n"
52 " of a line (not at any point in a line, like a comment in a shell script). \n"
53 " \n"
54 "In order to mimic the functionality of the OLDERADO tool mentioned above, one \n"
55 "can use the loos tool rmsds (or similar) to produce the matrix of similarity \n"
56 "scores. \n"
57 "For example: \n"
58 " \n"
59 "$ rmsds model.pdb ensemble.dcd | cluster-kgs -s > clustering_results.json \n"
60 " \n"
61 "would use rmsds to compute the alpha carbon RMSDs from the frame-pairs in \n"
62 "ensemble.dcd to generate the similarity matrix, then redirect it to cluster-\n"
63 "kgs, which will read from stdin because the -s flag was thrown. Then the \n"
64 "clustering results are written to an output file (which should be valid JSON, \n"
65 "for convenient further scripting). This shell-redirect would also cause the \n"
66 "distance matrix from rmsds to be written to stderr. Note that in this \n"
67 "particular command line the RMSD values emitted by rmsds will be in angstroms, \n"
68 "and will be rounded to 2 digits. For more reported precision (rmsds uses \n"
69 "doubles internally), use the '-p' flag. If you would like to both save the \n"
70 "similarities generated in this way, but also not have them written to disk \n"
71 "before feeding them to the clustering algorithm, you can redirect stderr and \n"
72 "stdout to separate files: \n"
73 " \n"
74 "$ rmsds model.pdb ensemble.dcd | \\\n"
75 " cluster-kgs -s 1> clustering_results.json 2>distances.asc \n"
76 " \n"
77 "You can also read a distance matrix from a file using the -f flag. If you do \n"
78 "that, it will not be emitted to stderr, and you would write: \n"
79 "cluster-kgs -f distances.asc > clustering_results.json \n"
80 " \n"
81 "The output from the clustering will be structured as JSON, and will have four \n"
82 "keys. One will be the invocation header, to help record where the data came \n"
83 "from. Then one recording which stage of the clustering was chosen as the cutoff\n"
84 " stage. One recording the penalty values at each stage (they could potentially \n"
85 "be quite similar). One will be a list (in no particular order) of the clusters,\n"
86 " where each cluster will be a list of element indexes corresponding to the \n"
87 "members of that cluster. Finally the last will be a list of exemplars for each \n"
88 "cluster. These will be in the same order as the list of clusters, so the first \n"
89 "exemplar in the list is the exemplar for the first cluster in the cluster list.\n"
90 ;
91  return helpstr;
92 }
